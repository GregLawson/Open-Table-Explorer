Bisection

  • Numerical Analysis

  • Git bisect command

  • Goldilocks the programmer and her three versions


Notes:

Bisection combines the divide-and-conquer and the successive refinement heuristics into a efficient search that makes minimal assumptions.

Bisection was originally a numerical analysis algorithm for finding x where f(x)=0. Initial conditions consist of an upper bound (too big) and a lower bound (too small). At each iteration the function is tested at the middle of the bounds, and one bound is replace by the tested middle value. Each iteration reduces the uncertainty by half. If there are multiple zeros, only one will be found.

Analogously git-bisect takes a bad commit descended from a good commit as the bounds and finds a bad commit directly followed by a good commit. Again if there are multiple bad/good transitions only one is found.
