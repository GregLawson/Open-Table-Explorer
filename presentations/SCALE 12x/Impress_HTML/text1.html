Short Abstract

Over the last thirty years neuro-scientists and psychologists have been studying the modularity of the brain, heuristics and biases.

Largely independently, software engineers have been developing methodologies and open source tools to use these defective brains to solve complex problems.

Git workflows, GUIs, unit testing, scripting, and bisection have been promising technologies, but too complex to be satisfying.

Software development is a process of separating our bugs from our enhancements so that we can appear to be geniuses, despite our failing tests proving we have defective brains.
